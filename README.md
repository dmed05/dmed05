## Featured projects

### [AI Workflow Builder](https://github.com/dmed05/ai-workflow-builder)

A portable three-file framework for defining an agent's behavior, operating
context, and approved references. Includes synthetic workflow examples,
publication guardrails, and automated secret scanning.

**Focus:** agent design · prompt architecture · reusable workflows · privacy

### [AI Inspection Report Demo](https://github.com/dmed05/ai-inspection-report-demo)

A secure synthetic inspection-report application with fixture-only public
routes and a protected reference API demonstrating owner isolation, restricted
CORS, rate limits, retention, and redacted logs.

**Focus:** Node.js · API security · test automation · responsible AI deployment

### [Local-First Agent Workflow](https://github.com/dmed05/local-first-agent-workflow)

A staged Python workflow engine that turns intake notes into a reviewed
deliverable while keeping source data local. It uses durable state, validation,
an append-only event trail, and a mandatory human approval gate.

**Focus:** Python · workflow orchestration · human-in-the-loop systems · CLI

### [Service Sales Operations Workflow](https://github.com/dmed05/service-sales-operations-workflow)

A fixture-only Node.js application that models inspection-led service sales as
an explicit state machine. Stage prerequisites, named human approval, and an
append-only audit trail control each handoff from intake through payment follow-up.

**Focus:** Node.js · state machines · operations workflows · privacy-by-design

## Engineering approach

- Start with a narrow problem and explicit definition of done.
- Keep production data out of public repositories and demos.
- Make security and privacy checks part of the development workflow.
- Separate AI generation from state, validation, and approval controls.
- Prefer small systems that can be tested and explained end to end.
